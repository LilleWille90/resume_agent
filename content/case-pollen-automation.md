# Case: Pollen-based automation for connected air purifiers

## Context
In Electrolux OneApp, we wanted to increase real-life value for users with allergies by making automation feel proactive and “set-and-forget”.

## Problem
Consumer-grade sensors can’t directly detect pollen, yet users still want their purifier to run when allergy risk is highest.

## Approach
- Combined external pollen data/APIs, geolocation and forecasts with on-device sensor data to create a meaningful trigger for automation
- Scoped tightly to ship fast and learn
- Shipped within two sprints and used product analytics to evaluate adoption/retention

## Result
- More than 25% of active users onboarded and kept the feature running continuously (strong signal of retained value)

## Why it worked
- Clear user problem
- Smart use of data + systems thinking
- Simple UX despite complex underlying logic

## Follow-up opportunities
- Better explainability (“why did it start now?”)
- Localisation per market
- Personalisation based on user behaviour and sensor baselines
